Git版本控制器
编辑时间：2017-07-25 00:03:19 作者：Mogo 浏览：0 评论：17

一、项目初始化

git init

git add *****文件

git commit -m "first commit"

git remote add origin http://192.168.35.195:3000/li6569049/laravel_project.git

git push -u origin master


二、查看当前分支

git branch 


三、查看所有分支

git branch -r


四、创建分支

git branch 分支名

git checkout -b 分支名 （此命令会创建且切换到该分支）


五、切换分支

git checkout 分支名 


六、提交到分支

git push -u origin 分支名


七、

在GIT 2.0中，simple将会是push.default的默认值。simple只会推送本地当前分支。


使用git config --global push.default simple  之后可直接使用git push上传



八、合并分支并提交

git merge 分支名

git push

在切换到mater   

git pull之后就是全新的模块


九、冲突解决

当提交有类似提示时  ![rejected] master -> master


第一步 git pull

提示有Auto-merging

查看冲突文件，显示那一段的代码有冲突

类似

1111111

<<<<<<< HEAD

3333333 // 这是本地的代码

========

22222222  //这边为服务器上的代码

>>>>>>>> 3414532453245745785667123


第二步

选择一个版本，见另一个版本删掉

此时可以git commit -m 了


十、查看当前的状态

git status


十一、查看版本

git commit


十二、查看所用的版本

git log


十三、切换版本

git checkout 版本的hash字段


十四、

比如：新来的同事，把别人的代码弄丢了。把本地的版本恢复到之前的版本。

git log

git reset --hard [commit编号]

git reflog 会列出当前版本之前的版本号
